# Security Policy

## Reporting a Vulnerability

Please report suspected vulnerabilities privately.

- Preferred channel: GitHub Security Advisory for this repository.
- Include reproduction details, impact assessment, and affected versions.
- Do not open public issues for unpatched vulnerabilities.

## Response Process

- Initial acknowledgement target: within 3 business days.
- Triage and severity classification after reproduction.
- Patch release and coordinated disclosure once fix is available.

## Scope

Security reports are welcome for:
- Arbitrary code execution paths.
- Unsafe file handling.
- Secret leakage through logs or reports.
- Supply chain risks in build and release workflows.
