# Support

## Getting Help

- Questions and usage help: GitHub Discussions.
- Bug reports: GitHub Issues.
- Security issues: follow `SECURITY.md`.

## Before Opening an Issue

1. Check existing issues and discussions.
2. Reproduce with latest `main`.
3. Provide command, config, and minimal fixture.
4. Include Python version and OS details.

## Maintainer Expectations

- Be respectful and precise.
- Keep reports focused on one problem at a time.
- Include expected and actual behavior.
